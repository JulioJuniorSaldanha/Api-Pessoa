PERGUNTA 1:
Durante a implementação de uma nova funcionalidade de software solicitada, quais critérios você avalia e implementa para garantia de qualidade de software?

R: Eu avalio que alem dessa funcionalidade fazer sentido para aplicação considere de extrema importancia que programador realize os testes da nova funcionalidade individualmente no ambiente local, e que tambem publique-a em ambiente de testes para ve o impaqto desta funcionalidade no sistema atual.
Para manter a qualidade do seu software tambem é de extrema importancia que o developer utilize conceitos como clean code que melhoram a visibilidade do código.
Mas acredito que o mais importante para avaliar se um software tem ou não qualidade são as seguintes: 
-As funções X estão disponíveis e são executadas eficientemente?
-Funciona corretamente em imprevistos, como, por exemplo, efetuar débito em uma conta com saldo negativo?
-O software é seguro, ou seja, evita que pessoas ou sistemas não autorizados tenham acesso aos dados para modificar?
-É fácil de usar ou requer muito treinamento?
-É fácil de integrar com outros sistemas existentes?
-Aceita trabalhar com arquivos de outros sistemas ou enviar dados para outros sistemas?

PERGUNTA 2:
Em qual etapa da implementação você considera a qualidade de software?

R: Para um software ter uma qualidade consistente ele deve des de a sua comcepção ser bem pensado por um PO, equipe de produtos e etc.pois todo o ciclo de vida de desenvolvimento ira se no modelo que foi pensado pelo PO.
por conta disso ha regras a serem seguidas para se desenvover um software de qualidade:
-Qualidade é competitividade.
-Qualidade é essencial para a sobrevivência.
-Qualidade é essencial para o mercado internacional.
-Qualidade é custo/benefício.
-Qualidade retém consumidores e aumenta lucros.

Mas claro que um software de qualidade deve ser bem escrito pelo developer, com facil manuseabilidade e facil manutenção.
E na etapa de testes o software deve tesr as caracteristicas que foram citadas na pergunta anterior
